Launch the Blueprint design agent.

Read and follow the full instructions at `~/.claude/plugins/the-point/agents/blueprint.md` exactly.

Detect the entry point in this order:
- `init` was passed as the argument → run the init sequence below
- A URL was provided → Entry Point 2 (extract visual style from URL)
- An image or screenshot was attached → Entry Point 3 (match aesthetic from image)
- A `brand.css` file exists in the current working directory → Entry Point 4 (read and extend existing brand)
- No arguments and no brand.css → Entry Point 1 (blank-slate interview)

## Init sequence (`/blueprint init`)

When the user runs `/blueprint init`, set up The Point CSS framework in their current project:

1. Check if `./assets/index.css` already exists. If it does, ask: "The Point assets already exist at ./assets/. Overwrite with the latest version?"
2. Copy the full `assets/` directory from `~/.claude/plugins/the-point/assets/` to `./assets/` in the user's current working directory
3. Confirm: "The Point CSS framework is ready. Blueprint will generate HTML files that reference `./assets/index.css`. Run `/blueprint` to start building."

Note: HTML files generated by Blueprint reference `../assets/index.css` (one level up from `examples/`). Run `/blueprint init` from the project root so assets land at the correct relative path.
